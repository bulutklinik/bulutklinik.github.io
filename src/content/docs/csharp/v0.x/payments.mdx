# Payments

`client.Payments` handles discount checks, saved cards and the appointment payment
entrypoint. All methods require a bearer token and return a `Task<JsonElement>`. The
amount is always computed **server-side** — you never send a price.

| Method                       | Endpoint |
|------------------------------|----------|
| `CheckDiscountCodeAsync(checkType, discountCode, …)`   | `POST /patients/checkDiscountCode` |
| `GetCardsAsync()`                 | `GET /payments/getCards` |
| `SaveCardAsync(card)`             | `POST /payments/saveCard` |
| `PayAsync(input)`                 | `POST /payments/interviewPayment` |
| `DeleteCardAsync(cardId)`         | `DELETE /payments/deleteCard/{cardId}` |

## `CheckDiscountCodeAsync`

Validate a discount code before paying. (Note: this endpoint lives under the
`patients` prefix, not `payments`.)

```csharp
var discount = await client.Payments.CheckDiscountCodeAsync(
    "appointment",  // checkType
    "WELCOME10",    // discountCode
    doctorId: 8282);
```

The full signature:

```csharp
Task<JsonElement> CheckDiscountCodeAsync(
    string checkType,
    string discountCode,
    object? doctorId = null,
    object? orderId = null,
    object? specialServiceId = null,
    string? programSlug = null,
    CancellationToken cancellationToken = default);
```

| Parameter          | Type      | Required | Notes |
|--------------------|-----------|:--------:|-------|
| `checkType`        | `string`  | ✓        | `"question"`, `"appointment"`, `"lab"`, `"special"`, `"physicallyAppointment"`, `"tmcLab"` or `"program"`. |
| `discountCode`     | `string`  | ✓        | The code to validate. |
| `doctorId`         | `object?` | ✓*       | Required except `lab` / `tmcLab` / `program`. |
| `orderId`          | `object?` | —        | Per check type. |
| `specialServiceId` | `object?` | —        | For `special`. |
| `programSlug`      | `string?` | —        | For `program`. |

A valid code returns an object with `discount_code`, `discount_title`,
`discount_id` and `prices`.

## Saved cards

```csharp
var cards = await client.Payments.GetCardsAsync();
// cards.GetProperty("cards") → [{ id, card_holder_name, card_number (masked), card_type, created_at }]

await client.Payments.SaveCardAsync(new CardInfo(
    CardHolder: "Ada Lovelace",
    CardNumber: "4111111111111111",
    CardExpMonth: "12",   // "MM"
    CardExpYear: "2030",  // "yyyy"
    CardCvv: "123"));

await client.Payments.DeleteCardAsync(cardId); // cardId from GetCardsAsync()
```

`CardInfo` is a positional record — all five members are required:

```csharp
public sealed record CardInfo(
    string CardHolder,
    string CardNumber,
    string CardExpMonth,
    string CardExpYear,
    string CardCvv);
```

`DeleteCardAsync(object cardId, …)` takes the id as `object`, so an `int`, `long` or
`string` all work.

## `PayAsync`

Start an appointment payment. Provide **either** a new `CardInfo` (all-or-none)
**or** a saved `CardId`.

```csharp
var result = await client.Payments.PayAsync(new PaymentInput
{
    DoctorId = 8282,
    AppointmentDate = "2026-06-20 14:30", // "yyyy-MM-dd HH:mm"
    AppointmentType = "interview",         // "interview" | "appointment"
    Is3D = true,
    TermsAccept = true,
    CardId = savedCardId,                  // …or CardInfo = new CardInfo(…)
    SaveCard = 0,                          // 1 tokenizes a new card
    DiscountCode = "WELCOME10",            // optional
});

if (result.TryGetProperty("payment3DUrl", out var urlEl))
{
    // Open this URL in a browser to complete 3-D Secure.
    string payment3DUrl = urlEl.GetString()!;
}
```

`PaymentInput`:

| Field             | Type        | Required | Default | Notes |
|-------------------|-------------|:--------:|---------|-------|
| `DoctorId`        | `object`    | ✓        | —       | `required` member. |
| `AppointmentDate` | `string` (`yyyy-MM-dd HH:mm`) | ✓ | — | `required` member. |
| `Is3D`            | `bool`      | —        | `false` | `true` returns a `payment3DUrl`. |
| `TermsAccept`     | `bool`      | —        | `false` | Must be `true`. |
| `AppointmentType` | `string`    | —        | `"interview"` | `"interview"` or `"appointment"`. |
| `CardInfo`        | `CardInfo?` | —*       | `null`  | A new card (all-or-none)… |
| `CardId`          | `object?`   | —*       | `null`  | …or a saved card id. |
| `SaveCard`        | `int`       | —        | `0`     | `1` tokenizes the card. |
| `DiscountCode`    | `string`    | —        | `""`    | Optional. |
| `CaseDetail`      | `string?`   | —        | `null`  | Opaque encrypted blob, passed through verbatim. |

## 3-D Secure

On a 3DS flow (`Is3D = true`) the result carries `payment3DUrl` — a **browser URL**
to open. The SDK returns it verbatim and does **not** follow, open or parse it. 3DS
completion (the "provizyon kapatma" / capture) happens browser ↔ bank ↔ server via a
callback, which is outside the SDK's scope. When `Is3D = false`, `data` is the
inline-completed order result with no `payment3DUrl`.

> `PayAsync` is rate-limited to 20 requests/hour/IP. On `Test`, it may return 404 if
> the POS isn't configured for the company; the SDK validates the request shape and
> the `payment3DUrl` return, not the bank capture.
