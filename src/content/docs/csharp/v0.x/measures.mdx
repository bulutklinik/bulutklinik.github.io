# Measures

`client.Measures` records and reads patient health measurements (tension, glucose,
weight, …). Patient endpoints require a bearer token; the partner endpoint uses the
configured partner token. Records are written to the authenticated patient. All
methods return a `Task<JsonElement>`.

| Method                              | Endpoint |
|-------------------------------------|----------|
| `AddListAsync(records)`                  | `POST /patients/addNewUserMeasures` |
| `AddAsync(measureType, fields)`          | `POST /patients/addNewUserMeasures/{type}` |
| `UpdateAsync(measureType, fields)`       | `PUT /patients/updateUserMeasures/{type}` |
| `DeleteAsync(measureType, id)`           | `DELETE /patients/deleteUserMeasures/{type}` |
| `LastAsync()`                            | `GET /patients/measuresList` |
| `ListAsync(measureType, page, glucoseType?)`    | `GET /patients/userMeasuresList/{type}/{page}/{glucoseType?}` |
| `GraphAsync(measureType, period, page, glucoseType?)` | `GET /patients/userMeasuresGraph/{type}/{period}/{page}/{glucoseType?}` |
| `PartnerHealthInformationAsync(identity, phoneNumber, data)`   | `POST /outher/healthInformation` |

Every measurement record also requires `date_time` as `"yyyy-MM-dd HH:mm"`. Records
and fields are plain `IDictionary<string, object?>` maps keyed by the wire field
names below.

## `AddListAsync(records)` — primary entrypoint

Submit one or many measurements (of any types) in a single transaction. Each record
is an `IDictionary<string, object?>`; the parameter is
`IEnumerable<IDictionary<string, object?>>`.

```csharp
await client.Measures.AddListAsync(new[]
{
    new Dictionary<string, object?>
    {
        ["type"] = "tension", ["hypertension"] = 120, ["hypotension"] = 80,
        ["date_time"] = "2026-06-19 09:00",
    },
    new Dictionary<string, object?>
    {
        ["type"] = "glucose", ["glucose"] = 95, ["glucose_type"] = 0,
        ["date_time"] = "2026-06-19 09:00",
    },
    new Dictionary<string, object?>
    {
        ["type"] = "weight", ["weight"] = 72.5, ["date_time"] = "2026-06-19 09:00",
    },
});
```

## `AddAsync` / `UpdateAsync` / `DeleteAsync`

Single-type writes. `AddAsync` / `UpdateAsync` take the type and an
`IDictionary<string, object?>` of fields; `DeleteAsync` takes the type and an
`object` id.

```csharp
// Add one measurement of a type.
await client.Measures.AddAsync("pulse", new Dictionary<string, object?>
{
    ["pulse"] = 68, ["date_time"] = "2026-06-19 09:00",
});

// Update an existing record (id required).
await client.Measures.UpdateAsync("pulse", new Dictionary<string, object?>
{
    ["id"] = 1234, ["pulse"] = 70, ["date_time"] = "2026-06-19 09:00",
});

// Delete a record (id required).
await client.Measures.DeleteAsync("pulse", 1234);
```

## Reading measurements

```csharp
// Latest value of every type.
var latest = await client.Measures.LastAsync();

// Paginated history for one type (page is 1-based; object so int or string works).
var history = await client.Measures.ListAsync("weight", 1);

// Glucose splits by glucoseType (0 = fasting, 1 = postprandial).
var glucose = await client.Measures.ListAsync("glucose", 1, 0);

// Grouped graph data. period: 1=day, 2=week, 3=month, 4=year.
var graph = await client.Measures.GraphAsync("tension", 3, 1);
```

The signatures:

```csharp
Task<JsonElement> ListAsync(string measureType, object page, int? glucoseType = null, CancellationToken cancellationToken = default);
Task<JsonElement> GraphAsync(string measureType, int period, object page, int? glucoseType = null, CancellationToken cancellationToken = default);
```

`LastAsync` returns the most recent of each type. Tension splits into
`hypertension` / `hypotension` and glucose into `hunger_glucose` /
`postprandial_glucose`, each with a `*Date`.

## Measure types

Use these `type` strings (passed as a plain `string`) with the listed wire fields.

| `type`    | Fields |
|-----------|--------|
| `tension` | `hypertension` (systolic), `hypotension` (diastolic) |
| `glucose` | `glucose`, `glucose_type` (0 = fasting, 1 = postprandial) |
| `pulse`   | `pulse` |
| `fever`   | `fever` |
| `weight`  | `weight` (BMI auto-computed) |
| `length`  | `length` (BMI auto-computed) |
| `waist`   | `waist` |
| `hip`     | `hip` |
| `fat`     | `fat` |
| `muscle`  | `muscle` |
| `calorie` | `calorie` |
| `step`    | `step` |
| `sleep`   | `sleep` (hours; stored to `sleep_time`) |

Values are numeric; `tension` / `pulse` are 1–10 digits, `glucose` is `0–99999.99`
and requires `glucose_type`, and weights/lengths are `0–99999.99`.

## Partner submission

`PartnerHealthInformationAsync` uses the partner (`teusan`) token configured via
`PartnerToken`, not the patient access token. The signature is
`Task<JsonElement> PartnerHealthInformationAsync(string? identity, string? phoneNumber, IEnumerable<IDictionary<string, object?>> data, CancellationToken cancellationToken = default)`.

```csharp
var partnerClient = new BulutklinikClient(new BulutklinikClientOptions
{
    PartnerToken = "…",
});

await partnerClient.Measures.PartnerHealthInformationAsync(
    "12345678901",            // identity
    "+90 555 111 22 33",      // phoneNumber
    new[]
    {
        new Dictionary<string, object?>
        {
            ["type"] = "tension", ["hypertension"] = 120, ["hypotension"] = 80,
            ["date_time"] = "2026-06-19 09:00",
        },
    });
```

> Known API behaviour: the partner endpoint has a server-side bug where `identity`
> is nulled during validation, so matching falls back to `phoneNumber`. The SDK
> sends the correct contract (`identity` + `phoneNumber`); send both.
