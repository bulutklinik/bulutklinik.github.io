# Payments

`client.Payments` handles discount checks, saved cards and the appointment payment
entrypoint. All methods require a bearer token and return the unwrapped `data` as a
`json.RawMessage`. The amount is always computed **server-side** — you never send a price.

| Method | Endpoint |
|--------|----------|
| `CheckDiscountCode(ctx, in)` | `POST /patients/checkDiscountCode` |
| `GetCards(ctx)` | `GET /payments/getCards` |
| `SaveCard(ctx, card)` | `POST /payments/saveCard` |
| `Pay(ctx, in)` | `POST /payments/interviewPayment` |
| `DeleteCard(ctx, cardID)` | `DELETE /payments/deleteCard/{cardID}` |

## `CheckDiscountCode(ctx, in)`

Validate a discount code before paying. (Note: this endpoint lives under the
`patients` prefix, not `payments`.)

```go
raw, err := client.Payments.CheckDiscountCode(ctx, bk.DiscountInput{
	CheckType:    "appointment",
	DiscountCode: "WELCOME10",
	DoctorID:     8282,
})
```

`DiscountInput`:

| Field              | Type     | Required | Notes |
|--------------------|----------|:--------:|-------|
| `CheckType`        | `string` | ✓        | `"question"`, `"appointment"`, `"lab"`, `"special"`, `"physicallyAppointment"`, `"tmcLab"` or `"program"`. |
| `DiscountCode`     | `string` | ✓        | The code to validate. |
| `DoctorID`         | `any`    | ✓*       | Required except `lab` / `tmcLab` / `program`. Omitted when `nil`. |
| `OrderID`          | `any`    | —        | Per check type. Omitted when `nil`. |
| `SpecialServiceID` | `any`    | —        | For `special`. Omitted when `nil`. |
| `ProgramSlug`      | `string` | —        | For `program`. Omitted when empty. |

A valid code returns `{ discount_code, discount_title, discount_id, prices }`.

## Saved cards

```go
raw, err := client.Payments.GetCards(ctx)
// data.cards: [{ id, card_holder_name, card_number (masked), card_type, created_at }]

_, err = client.Payments.SaveCard(ctx, bk.CardInfo{
	CardHolder:   "Ada Lovelace",
	CardNumber:   "4111111111111111",
	CardExpMonth: "12",   // "m"
	CardExpYear:  "2030", // "Y"
	CardCvv:      "123",
})

_, err = client.Payments.DeleteCard(ctx, cardID) // cardID from GetCards()
```

`CardInfo` fields (`CardHolder`, `CardNumber`, `CardExpMonth`, `CardExpYear`,
`CardCvv`) are all required. `cardID` is `any` — an `int` or `string` works.

## `Pay(ctx, in)`

Start an appointment payment. Provide **either** a new `CardInfo` (all-or-none, as a
pointer) **or** a saved `CardID`.

```go
raw, err := client.Payments.Pay(ctx, bk.PaymentInput{
	DoctorID:        8282,
	AppointmentDate: "2026-06-20 14:30", // "Y-m-d H:i"
	AppointmentType: "interview",         // "interview" | "appointment"
	Is3D:            true,
	TermsAccept:     true,
	CardID:          savedCardID,         // …or CardInfo: &bk.CardInfo{ … }
	SaveCard:        0,                    // 1 tokenizes a new card
	DiscountCode:    "WELCOME10",          // optional
})

var result struct {
	Payment3DURL string `json:"payment3DUrl"`
}
_ = json.Unmarshal(raw, &result)
if result.Payment3DURL != "" {
	// Open this URL in a browser to complete 3-D Secure.
}
```

`PaymentInput`:

| Field             | Type        | Required | Notes |
|-------------------|-------------|:--------:|-------|
| `DoctorID`        | `any`       | ✓        | An `int` or `string`. |
| `AppointmentDate` | `string`    | ✓        | `Y-m-d H:i`. |
| `Is3D`            | `bool`      | ✓        | `true` returns a `payment3DUrl`. |
| `TermsAccept`     | `bool`      | ✓        | Must be `true`. |
| `AppointmentType` | `string`    | —        | `"interview"` or `"appointment"`. Defaults to `"interview"` when empty. |
| `CardInfo`        | `*CardInfo` | —*       | A new card (all-or-none)… |
| `CardID`          | `any`       | —*       | …or a saved card id. |
| `SaveCard`        | `int`       | —        | `1` tokenizes the card. |
| `DiscountCode`    | `string`    | —        | Optional. |
| `CaseDetail`      | `string`    | —        | Opaque encrypted blob, passed through verbatim. Omitted when empty. |

## 3-D Secure

On a 3DS flow (`Is3D: true`) the result carries `payment3DUrl` — a **browser URL**
to open. The SDK returns it verbatim and does **not** follow, open or parse it.
3DS completion (the "provizyon kapatma" / capture) happens browser ↔ bank ↔ server
via a callback, which is outside the SDK's scope. When `Is3D: false`, `data` is the
inline-completed order result with no `payment3DUrl`.

> `Pay` is rate-limited to 20 requests/hour/IP. On `Test`, it may return 404 if the
> POS isn't configured for the company; the SDK validates the request shape and the
> `payment3DUrl` return, not the bank capture.
