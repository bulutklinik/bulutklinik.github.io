# Payments

`client.payments` handles discount checks, saved cards and the appointment payment
entrypoint. All methods require a bearer token. The amount is always computed
**server-side** — you never send a price.

| Method                       | Endpoint |
|------------------------------|----------|
| `checkDiscountCode(input)`   | `POST /patients/checkDiscountCode` |
| `getCards()`                 | `GET /payments/getCards` |
| `saveCard(card)`             | `POST /payments/saveCard` |
| `pay(input)`                 | `POST /payments/interviewPayment` |
| `deleteCard(cardId)`         | `DELETE /payments/deleteCard/{cardId}` |

## `checkDiscountCode(input)`

Validate a discount code before paying. (Note: this endpoint lives under the
`patients` prefix, not `payments`.)

```ts
const discount = await client.payments.checkDiscountCode({
  checkType: "appointment",
  discountCode: "WELCOME10",
  doctorId: 8282,
});
```

`DiscountCheckInput`:

| Field              | Type                | Required | Notes |
|--------------------|---------------------|:--------:|-------|
| `checkType`        | `DiscountCheckType` | ✓        | `"question" \| "appointment" \| "lab" \| "special" \| "physicallyAppointment" \| "tmcLab" \| "program"`. |
| `discountCode`     | `string`            | ✓        | The code to validate. |
| `doctorId`         | `number \| string`  | ✓*       | Required except `lab` / `tmcLab` / `program`. |
| `orderId`          | `number \| string`  | —        | Per check type. |
| `specialServiceId` | `number \| string`  | —        | For `special`. |
| `programSlug`      | `string`            | —        | For `program`. |

A valid code returns `{ discount_code, discount_title, discount_id, prices }`.

## Saved cards

```ts
const { cards } = await client.payments.getCards();
// cards: [{ id, card_holder_name, card_number (masked), card_type, created_at }]

await client.payments.saveCard({
  cardHolder: "Ada Lovelace",
  cardNumber: "4111111111111111",
  cardExpMonth: "12", // "m"
  cardExpYear: "2030", // "Y"
  cardCvv: "123",
});

await client.payments.deleteCard(cardId); // cardId from getCards()
```

`CardInfo` fields (`cardHolder`, `cardNumber`, `cardExpMonth`, `cardExpYear`,
`cardCvv`) are all required.

## `pay(input)`

Start an appointment payment. Provide **either** a new `cardInfo` (all-or-none)
**or** a saved `cardId`.

```ts
const result = await client.payments.pay({
  doctorId: 8282,
  appointmentDate: "2026-06-20 14:30", // "Y-m-d H:i"
  appointmentType: "interview",         // "interview" | "appointment"
  is3D: true,
  termsAccept: true,
  cardId: savedCardId,                  // …or cardInfo: { … }
  saveCard: 0,                          // 1 tokenizes a new card
  discountCode: "WELCOME10",            // optional
});

if (result.payment3DUrl) {
  // Open this URL in a browser to complete 3-D Secure.
  window.location.href = result.payment3DUrl;
}
```

`PaymentInput`:

| Field             | Type                            | Required | Notes |
|-------------------|---------------------------------|:--------:|-------|
| `doctorId`        | `number \| string`              | ✓        | — |
| `appointmentDate` | `string` (`Y-m-d H:i`)          | ✓        | — |
| `is3D`            | `boolean`                       | ✓        | `true` returns a `payment3DUrl`. |
| `termsAccept`     | `boolean`                       | ✓        | Must be accepted. |
| `appointmentType` | `"interview" \| "appointment"`  | —        | Default `"interview"`. |
| `cardInfo`        | `CardInfo`                      | —*       | A new card (all-or-none)… |
| `cardId`          | `number \| string`              | —*       | …or a saved card id. |
| `saveCard`        | `0 \| 1`                        | —        | `1` tokenizes the card. |
| `discountCode`    | `string`                        | —        | Optional. |
| `caseDetail`      | `string`                        | —        | Opaque encrypted blob, passed through verbatim. |

## 3-D Secure

On a 3DS flow (`is3D: true`) the result carries `payment3DUrl` — a **browser URL**
to open. The SDK returns it verbatim and does **not** follow, open or parse it.
3DS completion (the "provizyon kapatma" / capture) happens browser ↔ bank ↔ server
via a callback, which is outside the SDK's scope. When `is3D: false`, `data` is the
inline-completed order result with no `payment3DUrl`.

> `pay` is rate-limited to 20 requests/hour/IP. On `test`, it may return 404 if the
> POS isn't configured for the company; the SDK validates the request shape and the
> `payment3DUrl` return, not the bank capture.
