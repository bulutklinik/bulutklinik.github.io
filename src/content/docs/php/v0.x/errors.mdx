# Errors & responses

## The response envelope

Every API response is a JSON envelope. The SDK unwraps it for you — on success it
returns the `data` payload directly (as a PHP value/array); on failure it throws a
typed exception.

```jsonc
{
  "resultType": 0,            // state code (see below)
  "errorType": "validation",  // string label OR numeric code
  "errorMessage": "…",        // human-readable, localized via `lang`
  "successMessage": "…",
  "data": { /* payload */ }   // what the SDK returns on success
}
```

A call is successful only when the HTTP status is 2xx **and** `resultType === 0`.
The `resultType` state codes:

| Value | Name      | Meaning |
|:-----:|-----------|---------|
| `0`   | `Success` | Return `data`. |
| `1`   | `Error`   | Throw `ApiException` (or a subtype). |
| `2`   | `Logout`  | Session revoked — token store cleared, `AuthenticationException`. |
| `3`   | `Update`  | Client/app too old — `ApiException`. |
| `4`   | `Refresh` | Token expired — triggers silent refresh + retry. |

> `errorType` is **polymorphic**: some endpoints return a string label
> (`"validation"`), others a numeric code. The SDK keeps it typed as
> `string|int|null` and treats it as a refinement hint only when it is a string.

## Exception hierarchy

All exceptions extend `BulutklinikException` (which extends `\RuntimeException`):

```
BulutklinikException              (base — all SDK exceptions)
├── TransportException            (network failure, timeout, DNS, TLS — no HTTP response)
└── ApiException                  (an HTTP response that wasn't a success)
    ├── ValidationException       (422, or errorType "validation")
    ├── AuthenticationException   (401 / logout / failed refresh)
    ├── AuthorizationException    (403)
    ├── NotFoundException         (404)
    └── RateLimitException        (429 — carries retryAfter)
```

Every `ApiException` carries a readonly `$context` of type `ApiErrorContext` with
the fields for debugging:

| Property      | Type                  | Notes |
|---------------|-----------------------|-------|
| `httpStatus`  | `int`                 | The HTTP status code. |
| `resultType`  | `?int`                | Envelope `resultType`, when present. |
| `errorType`   | `string\|int\|null`   | Polymorphic label or code. |
| `data`        | `mixed`               | Raw envelope `data`. |
| `method`      | `?string`             | Originating HTTP method. |
| `path`        | `?string`             | Originating request path. |
| `retryAfter`  | `?int`                | Seconds from the `Retry-After` header (on `RateLimitException`). |

```php
<?php

use Bulutklinik\Sdk\Exception\ApiException;
use Bulutklinik\Sdk\Exception\AuthenticationException;
use Bulutklinik\Sdk\Exception\RateLimitException;
use Bulutklinik\Sdk\Exception\TransportException;
use Bulutklinik\Sdk\Exception\ValidationException;

try {
    $client->payments->pay(
        doctorId: 8282,
        appointmentDate: '2026-06-20 14:30',
        is3D: true,
        termsAccept: true,
    );
} catch (RateLimitException $e) {
    fwrite(STDERR, "throttled — retry after {$e->context->retryAfter}s\n");
} catch (ValidationException $e) {
    var_dump($e->context->data);
} catch (AuthenticationException $e) {
    fwrite(STDERR, "session expired — re-login required\n");
} catch (TransportException $e) {
    fwrite(STDERR, 'network problem: ' . $e->getMessage() . "\n");
} catch (ApiException $e) {
    fwrite(STDERR, "API error {$e->context->httpStatus} on {$e->context->method} {$e->context->path}\n");
}
```

> `TransportException` extends `BulutklinikException` directly (not `ApiException`),
> so it has **no** `$context` — its details come from `getMessage()` and the
> previous exception via `getPrevious()`. Only `ApiException` and its subtypes carry
> `$context`.

## Mapping precedence

The SDK picks the most specific exception type in this order
(`ApiException::fromContext`):

1. `resultType === 2` (logout) → `AuthenticationException`
2. string `errorType === "validation"` (case-insensitive) or HTTP 422 → `ValidationException`
3. HTTP status: 401 → `AuthenticationException`, 403 → `AuthorizationException`,
   404 → `NotFoundException`, 429 → `RateLimitException`
4. otherwise (including numeric `errorType`, or 2xx with `resultType !== 0`) → `ApiException`

Because `errorType` may be numeric, the SDK guards before string-matching it — and
so should any code that inspects `$e->context->errorType`.
