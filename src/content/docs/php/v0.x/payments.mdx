# Payments

`$client->payments` handles discount checks, saved cards and the appointment payment
entrypoint. All methods require a bearer token. The amount is always computed
**server-side** — you never send a price.

| Method                                                                 | Endpoint |
|------------------------------------------------------------------------|----------|
| `checkDiscountCode(checkType, discountCode, doctorId?, orderId?, specialServiceId?, programSlug?)` | `POST /patients/checkDiscountCode` |
| `getCards()`                                                           | `GET /payments/getCards` |
| `saveCard(cardHolder, cardNumber, cardExpMonth, cardExpYear, cardCvv)` | `POST /payments/saveCard` |
| `pay(doctorId, appointmentDate, is3D, termsAccept, …)`                 | `POST /payments/interviewPayment` |
| `deleteCard(cardId)`                                                   | `DELETE /payments/deleteCard/{cardId}` |

## `checkDiscountCode(...)`

Validate a discount code before paying. (Note: this endpoint lives under the
`patients` prefix, not `payments`.)

```php
$discount = $client->payments->checkDiscountCode(
    checkType: 'appointment',
    discountCode: 'WELCOME10',
    doctorId: 8282,
);
```

`checkDiscountCode` parameters:

| Parameter          | Type                  | Required | Notes |
|--------------------|-----------------------|:--------:|-------|
| `checkType`        | `string`              | ✓        | `question` \| `appointment` \| `lab` \| `special` \| `physicallyAppointment` \| `tmcLab` \| `program`. |
| `discountCode`     | `string`              | ✓        | The code to validate. |
| `doctorId`         | `int\|string\|null`   | ✓*       | Required except `lab` / `tmcLab` / `program`. |
| `orderId`          | `int\|string\|null`   | —        | Per check type. |
| `specialServiceId` | `int\|string\|null`   | —        | For `special`. |
| `programSlug`      | `?string`             | —        | For `program`. |

A valid code returns `['discount_code' => …, 'discount_title' => …, 'discount_id' => …, 'prices' => …]`.

## Saved cards

```php
$result = $client->payments->getCards();
// $result['cards']: [['id' => …, 'card_holder_name' => …, 'card_number' => (masked), 'card_type' => …, 'created_at' => …], …]

$client->payments->saveCard(
    cardHolder: 'Ada Lovelace',
    cardNumber: '4111111111111111',
    cardExpMonth: '12',   // "m"
    cardExpYear: '2030',  // "Y"
    cardCvv: '123',
);

$client->payments->deleteCard($cardId); // cardId from getCards()
```

`saveCard` parameters (`cardHolder`, `cardNumber`, `cardExpMonth`, `cardExpYear`,
`cardCvv`) are all required `string`s. `deleteCard` takes a single `int|string $cardId`.

## `pay(...)`

Start an appointment payment. Provide **either** a new `cardInfo` (all-or-none)
**or** a saved `cardId`.

```php
$result = $client->payments->pay(
    doctorId: 8282,
    appointmentDate: '2026-06-20 14:30', // "Y-m-d H:i"
    is3D: true,
    termsAccept: true,
    appointmentType: 'interview',        // "interview" | "appointment"
    cardId: $savedCardId,                // …or cardInfo: [...]
    saveCard: 0,                         // 1 tokenizes a new card
    discountCode: 'WELCOME10',           // optional
);

if (isset($result['payment3DUrl'])) {
    // Open this URL in a browser to complete 3-D Secure.
    header('Location: ' . $result['payment3DUrl']);
}
```

`pay` parameters:

| Parameter         | Type                  | Required | Default | Notes |
|-------------------|-----------------------|:--------:|---------|-------|
| `doctorId`        | `int\|string`         | ✓        | —       | — |
| `appointmentDate` | `string` (`Y-m-d H:i`)| ✓        | —       | — |
| `is3D`            | `bool`                | ✓        | —       | `true` returns a `payment3DUrl`. |
| `termsAccept`     | `bool`                | ✓        | —       | Must be accepted. |
| `appointmentType` | `string`              | —        | `'interview'` | `interview` \| `appointment`. |
| `cardInfo`        | `?array`              | —*       | `null`  | A new card (all-or-none)… |
| `cardId`          | `int\|string\|null`   | —*       | `null`  | …or a saved card id. |
| `saveCard`        | `int`                 | —        | `0`     | `1` tokenizes the card. |
| `discountCode`    | `string`              | —        | `''`    | Optional. |
| `caseDetail`      | `?string`             | —        | `null`  | Opaque encrypted blob, passed through verbatim. |

`cardInfo` is an associative array with the keys `cardHolder`, `cardNumber`,
`cardExpMonth`, `cardExpYear`, `cardCvv` (all `string`).

## 3-D Secure

On a 3DS flow (`is3D: true`) the result carries `payment3DUrl` — a **browser URL**
to open. The SDK returns it verbatim and does **not** follow, open or parse it.
3DS completion (the "provizyon kapatma" / capture) happens browser ↔ bank ↔ server
via a callback, which is outside the SDK's scope. When `is3D: false`, `data` is the
inline-completed order result with no `payment3DUrl`.

> `pay` is rate-limited to 20 requests/hour/IP. On `test`, it may return 404 if the
> POS isn't configured for the company; the SDK validates the request shape and the
> `payment3DUrl` return, not the bank capture.
